# 词语与句子收集册设计规划

## 1. 目标

这份文档用于规划应用内的“词语收集册 / 句子收集册”功能，目标是把当前的“看视频、点句解析、实时字幕学习”进一步延伸为“可沉淀、可复习、可导出”的学习系统。

核心价值：

- 把即时学习内容沉淀为长期记忆素材
- 让用户可以把“关键词”和“例句”按主题、语言、难度分类保存
- 让点句解析结果不再只是一次性阅读，而是可反复复习的学习资产
- 为后续的复习、测验、卡片化记忆、导出打印或移动端同步做准备

## 2. 功能范围

本期规划聚焦四类能力：

1. 收藏词语
2. 收藏句子
3. 管理收集册
4. 导出收集内容

暂不包含但需要为后续预留的数据能力：

- 间隔重复复习
- 掌握度统计
- 生词测验
- 云同步
- 多设备共享

## 3. 产品结构

建议新增一个独立模块：`收集册`

模块下分两类实体：

- `词语收集册`
- `句子收集册`

同时支持“册子”概念：

- 一个用户可以创建多个词语册
- 一个用户可以创建多个句子册
- 后续也可以演进为“混合收集册”，但第一版建议先分开

推荐的顶层导航结构：

- 视频学习台
- 视频库
- 收集册
- 设置

## 4. 用户故事

### 4.1 词语收藏

用户在“点句解析 / 高级翻译”里看到关键词卡片时：

- 点击某个关键词
- 弹出菜单或小面板
- 选择“收藏到词语册”
- 选择已有词语册，或新建一个词语册
- 保存后显示“已收藏”

用户保存的词语信息应尽量完整，而不只是一个孤立单词。

### 4.2 句子收藏

用户在“字幕总览”里看到某一句时：

- 点击句子卡片的收藏按钮
- 或在点句解析区对当前句子点击“收藏句子”
- 选择“收藏到句子册”
- 选择已有句子册，或新建一个句子册

句子收藏时，可以附带它对应的点句解析结果。

### 4.3 管理册子

用户可以：

- 创建册子
- 重命名册子
- 删除册子
- 查看册子内条目数量
- 按语言、视频来源、收藏时间筛选

### 4.4 导出

用户可以导出：

- 单个词语册
- 单个句子册
- 多个册子合并导出

推荐优先支持：

- `CSV`
- `JSON`
- `Markdown`

后续可扩展：

- `XLSX`
- 可打印 PDF
- Anki 导入格式

## 5. 数据来源

### 5.1 词语来源

词语仅来自“点句解析 / 高级翻译”的 `keywords`

当前每个关键词已有这些字段：

- `word`
- `meaning`
- `note`

但收藏时建议补充更多上下文：

- 所属句子原文
- 所属句子译文
- 所属视频
- 句子时间戳
- 分析模型名称
- 收藏时的源语言 / 学习语言 / 母语

### 5.2 句子来源

句子来自：

- 字幕总览中的任意一条字幕
- 当前点句解析区的当前选中句子

句子收藏时建议附带：

- 原文句子
- 译文句子
- 时间范围
- 所属视频
- 源语言 / 学习语言 / 母语
- 对应点句解析结果

## 6. 第一版交互设计

## 6.1 关键词交互

在关键词卡片上增加一个操作入口，推荐两种方案：

方案 A：关键词卡片右上角加 `收藏` 小按钮

- 优点：清晰直接
- 缺点：卡片密集时可能显得拥挤

方案 B：点击整个关键词卡片后弹出操作面板

- 选项：`收藏到词语册`
- 选项：`复制词语`
- 选项：`查看上下文`

第一版更推荐方案 A，因为学习动作更直接。

### 6.2 句子交互

在字幕总览的每个句子卡片上增加：

- `收藏句子` 图标按钮

在点句解析区顶部的当前句子卡片上增加：

- `收藏当前句子` 按钮

推荐两处都保留：

- 字幕总览适合快速扫读收藏
- 点句解析区适合先看懂再收藏

### 6.3 选择册子

当用户点击收藏时，弹出轻量面板：

- 最近使用的册子
- 所有可选册子
- 新建册子输入框

建议支持：

- 记住最近一次选择
- 连续收藏时默认选中最近册子

这样可以减少重复操作。

## 7. 收集条目数据结构

第一版建议用 SQLite 持久化。

### 7.1 表：`notebooks`

字段建议：

- `id`
- `type`：`word` / `sentence`
- `name`
- `description`
- `source_lang`
- `learning_lang`
- `native_lang`
- `created_at`
- `updated_at`

说明：

- 册子可以有语言维度，但不强制
- 如果不限制语言，则对应字段可为空

### 7.2 表：`word_entries`

字段建议：

- `id`
- `notebook_id`
- `word`
- `meaning`
- `note`
- `source_sentence`
- `learning_sentence`
- `segment_id`
- `video_id`
- `video_stem`
- `start_time`
- `end_time`
- `analysis_model`
- `analysis_payload_json`
- `created_at`
- `updated_at`

说明：

- `analysis_payload_json` 用于保留收藏当时的完整解析快照
- 后续即使视频重新翻译，已收藏内容也不会丢失

### 7.3 表：`sentence_entries`

字段建议：

- `id`
- `notebook_id`
- `source_text`
- `learning_text`
- `source_lang`
- `learning_lang`
- `native_lang`
- `segment_id`
- `video_id`
- `video_stem`
- `start_time`
- `end_time`
- `analysis_model`
- `analysis_payload_json`
- `created_at`
- `updated_at`

说明：

- 句子条目建议保存完整分析结果
- 这样在收集册中可以直接查看当时的优化译文、语法点、关键词、追问建议

### 7.4 去重策略

第一版建议“软去重”：

- 同一个册子内，相同 `word + source_sentence + video_id` 视为重复
- 同一个册子内，相同 `segment_id + video_id` 的句子视为重复

重复时：

- 不再新增
- 更新 `updated_at`
- 前端提示“已在该册中”

- `GET /api/notebooks`

### 8.4 导出

- `GET /api/notebooks/{id}/export?format=csv`
- `GET /api/notebooks/{id}/export?format=json`
- `GET /api/notebooks/{id}/export?format=md`

## 9. 前端页面规划

## 9.1 新页面：收集册主页

布局建议：

- 左侧：册子列表
- 右侧：当前册子内容

左侧展示：

- 词语册分组
- 句子册分组
- 每个册子的条目数量

右侧展示：

- 册子标题
- 过滤器
- 排序方式
- 导出按钮
- 条目列表

## 9.2 词语册详情页

每个词语条目建议展示：

- 词语原文
- 中文释义
- 学习备注
- 对应原句
- 对应译文
- 来源视频
- 收藏时间

可选操作：

- 删除
- 复制
- 跳转回原视频位置

## 9.3 句子册详情页

每个句子条目建议展示：

- 原句
- 译文
- 时间戳
- 来源视频
- 可展开查看解析结果

展开后显示：

- 优化译文
- 自然译文
- 句子结构
- 关键词
- 语法点
- 继续追问

## 10. 与当前系统的衔接

### 10.1 和点句解析模块的关系

词语收藏直接依赖点句解析结果，因此需要保证：

- 当分析未完成时，关键词收藏按钮不可用
- 当分析已完成时，关键词卡片可直接收藏
- 若分析结果来自缓存，也应支持收藏

### 10.2 和字幕总览模块的关系

句子收藏需要从字幕总览中拿到句子信息，因此建议：

- 每条字幕卡片增加收藏入口
- 收藏后不打断当前播放
- 仅做轻量提示反馈

### 10.3 和视频跳转的关系

后续建议支持：

- 在收集册中点击句子或词语
- 跳回原视频
- 自动定位到对应时间点

这会让收集册从“静态列表”升级为“可回看上下文”的学习工具。

### 11.1 词语册 CSV

推荐列：

- `word`
- `meaning`
- `note`
- `source_sentence`
- `learning_sentence`
- `video_title`
- `start_time`
- `created_at`

### 11.2 句子册 CSV

推荐列：

- `source_text`
- `learning_text`
- `video_title`
- `start_time`
- `end_time`
- `created_at`

### 11.3 JSON

JSON 导出建议保留完整结构：

- 册子元数据
- 所有条目
- 分析快照

这样便于后续导入、备份、迁移。

### 11.4 Markdown

Markdown 导出适合：

- 打印学习
- 手动整理笔记
- 发到 Obsidian、Notion、语雀等笔记系统

## 12. 第一版实施建议

建议分三步做：

### 阶段 1：最小可用版

- 新增数据库表
- 支持创建词语册 / 句子册
- 支持从关键词收藏到词语册
- 支持从字幕句子收藏到句子册
- 支持基础列表查看

### 阶段 2：完善查看体验

- 句子册内展开查看解析结果
- 支持跳回视频
- 支持搜索、筛选、排序
- 支持去重提示

### 阶段 3：导出与复习增强

- CSV / JSON / Markdown 导出
- 最近收藏
- 按视频聚合
- 预留复习状态字段

## 13. 风险与注意点

### 13.1 分析结果可能变化

如果用户重新调用不同模型重新解析同一句，结果可能不同。

建议第一版策略：

- 收藏时保存“当时的解析快照”
- 不自动覆盖旧收藏内容

### 13.2 关键词格式不稳定

不同模型输出的关键词数量、粒度可能不同。

建议第一版策略：

- 只收藏当前已经展示出来的关键词项
- 不在收藏时再做二次拆词

### 13.3 视频删除后的数据一致性

如果视频被从视频库删除：

- 收集册条目仍应保留
- 但标注“来源视频已删除”

因此收藏条目不应只依赖视频外键，还应冗余保存视频标题和句子文本。

## 14. 建议结论

本功能建议正式纳入产品路线，定位为：

- 视频学习台的“沉淀层”
- 点句解析的“记忆层”
- 后续复习系统的“数据基础层”

第一版推荐优先级：

1. 册子管理
2. 关键词收藏
3. 句子收藏
4. 收集册查看
5. 导出

这样可以尽快把“学完即走”的体验，升级成“学完可积累、可回顾、可整理”的完整闭环。
