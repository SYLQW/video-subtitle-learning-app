# 多语言学习模式与字幕导出设计

## 目标

这份文档记录以下产品方向：

- 视频支持多语言源语言输入，而不只限于英语
- 用户可以指定“学习目标语言”
- 用户可以指定自己的“解释语言/母语”
- 应用支持多种字幕显示模式
- 应用支持导出字幕文件与导出带字幕视频

这套设计的目标不是只服务“英语视频 -> 中文字幕”这一种场景，而是扩展为：

- 任意源语言内容
- 任意学习目标语言
- 任意解释语言

从而支持更多国家和地区的学习者。

## 核心概念

### 1. 源语言

源语言指视频原始语音所属语言。

例如：

- 英语视频 -> 源语言是英语
- 日语视频 -> 源语言是日语
- 韩语视频 -> 源语言是韩语

系统应支持：

- 自动检测
- 手动指定

### 2. 学习目标语言

学习目标语言指用户想重点学习的语言。

例如：

- 中国用户看英语视频，学习目标语言是英语
- 日本视频转成英语学习材料，学习目标语言是英语
- 英语视频转成日语学习材料，学习目标语言是日语

学习目标语言会影响：

- 字幕翻译目标
- 点句解析重点
- 高级翻译输出方向
- 导出字幕内容

### 3. 解释语言 / 母语

解释语言指用户希望看到的解释、语法说明、学习建议所使用的语言。

例如：

- 中国用户：解释语言是中文
- 日本用户：解释语言是日语
- 法语用户：解释语言是法语

解释语言会影响：

- 点句解析提示词
- 语法解释提示词
- 高级翻译说明文字
- 继续追问建议语言

### 4. 显示模式

显示模式决定当前字幕区域如何展示文本。

建议支持：

- 单语
- 双语
- 三语

具体展示内容由源语言、学习目标语言、解释语言共同决定。

## 推荐处理流程

### 标准链路

推荐采用：

1. 语音识别得到源语言文本
2. 保留源语言原文
3. 将源语言文本翻译为学习目标语言
4. 如有需要，再生成解释语言版本的补充说明
5. 前端根据显示模式决定展示哪些文本

### 为什么不建议直接只保留译文

如果只保留译文，会损失：

- 原句对照能力
- 多语言切换能力
- 后续重新翻译到其他目标语言的能力
- 更精细的点句解析能力

因此应优先保存：

- `source_text`
- `learning_text`
- `native_explanation_text` 或解释结果

## 典型场景

### 场景 A：英语视频，中文用户学英语

- 源语言：英语
- 学习目标语言：英语
- 解释语言：中文

推荐展示：

- 双语：英语 + 中文
- 点句解析：以中文解释英语句子

### 场景 B：日语视频，中文用户学英语

- 源语言：日语
- 学习目标语言：英语
- 解释语言：中文

推荐展示：

- 单语：英语
- 双语：日语 + 英语
- 三语：日语 + 英语 + 中文说明

点句解析应重点分析：

- 英语表达
- 日语原句与英语表达的对应关系
- 用中文说明学习要点

### 场景 C：韩语视频，日本用户学英语

- 源语言：韩语
- 学习目标语言：英语
- 解释语言：日语

推荐展示：

- 双语：韩语 + 英语
- 点句解析：用日语解释英语表达

## 字幕模式设计

### 应用内字幕显示

应用内字幕总览和播放器联动区域应支持：

- 源语言字幕
- 学习目标语言字幕
- 双语字幕
- 三语字幕

在设置页中建议拆成两个独立维度：

- 字幕内容模式
- 解释语言

### 推荐显示模式

建议支持以下模式：

1. `source_only`
   - 仅显示源语言
2. `learning_only`
   - 仅显示学习目标语言
3. `source_learning`
   - 源语言 + 学习目标语言
4. `source_learning_native`
   - 源语言 + 学习目标语言 + 解释语言摘要

## 导出能力

### 已确认支持的导出方向

本项目应同时支持：

- 导出英文字幕版
- 导出中文字幕版
- 导出双语字幕版
- 导出“带字幕轨的视频”
- 导出“烧录版视频”

### 字幕文件导出

建议支持导出：

- `source.srt`
- `learning.srt`
- `native.srt`（如适用）
- `bilingual.srt`
- `trilingual.srt`（如适用）

### 视频导出

建议支持两类视频导出：

#### 1. 软字幕视频

特点：

- 优先推荐
- 不改变原视频画面
- 字幕轨可开关
- 可提供多条字幕轨

适合：

- 应用内管理
- 多语言切换
- 保留原画质

#### 2. 烧录版视频

特点：

- 字幕直接嵌入画面
- 所有播放器都可见
- 无法关闭字幕

适合：

- 分享给其他人
- 移动设备或兼容性要求高的环境

### 软字幕导出策略

用户已明确倾向于：

- 在应用内优先用软字幕
- 由程序提供轨道选择

因此正式设计中应将软字幕视频作为默认导出方案。

建议支持导出轨道：

- 仅源语言轨
- 仅学习目标语言轨
- 双语轨

## FFmpeg 导出策略

### 软字幕

技术上建议使用：

- `SRT` 或 `ASS`
- 封装到 `MKV` 或适配场景的 `MP4`

软字幕适合多轨道场景，因为：

- 可以保留多个语言轨
- 可以在播放器中切换

### 烧录字幕

技术上建议使用：

- FFmpeg `subtitles` 或 `ass` 滤镜

烧录时建议允许用户选择：

- 单语
- 双语
- 三语

## 转写与翻译策略

### faster-whisper 的角色

`faster-whisper` 的主要职责应是：

- 识别语音
- 得到源语言文本
- 可选自动检测源语言

### 翻译策略

推荐优先采用：

- 先保留源语言文本
- 再翻译到学习目标语言

而不是把整个系统绑定成：

- 英语 -> 中文

### 为什么这样设计更好

因为这样可以支持：

- 日语 -> 英语
- 韩语 -> 英语
- 英语 -> 日语
- 西语 -> 英语

同一份源文本还能重复生成不同学习目标语言版本。

## 大模型提示词与翻译服务联动

### 必须调整的参数

如果系统支持国际化学习场景，那么下列内容不能写死：

- 翻译目标语言
- 点句解析使用语言
- 高级翻译说明使用语言
- 提问建议语言

### 提示词应包含的变量

分析与翻译提示词建议显式包含：

- `source_language`
- `learning_language`
- `native_language`
- `display_mode`

### 例子

例如：

- 源语言：日语
- 学习目标语言：英语
- 解释语言：中文

则提示词应要求模型：

- 输出学习目标语言译文为英语
- 语法分析和解释使用中文
- 必要时说明源语言与英语表达的差异

### 对翻译服务的要求

如果使用翻译服务而不是通用大模型，也必须支持动态指定：

- `source_lang`
- `target_lang`

不能写死为：

- `EN -> ZH`

## 数据结构建议

单条字幕片段未来建议至少保留：

- `id`
- `start`
- `end`
- `source_lang`
- `source_text`
- `learning_lang`
- `learning_text`
- `native_lang`
- `native_text` 或 `native_summary`

如果后续支持多次生成不同目标语言版本，建议改成：

- 源文本一份
- 多个目标语言翻译版本
- 多个解释语言解析版本

## 设置页建议

建议新增以下设置项：

- 源语言：自动 / 手动指定
- 学习目标语言
- 解释语言 / 母语
- 字幕显示模式
- 默认导出模式

### 导出设置建议

用户导出时应可选择：

- 导出字幕文件
- 导出带字幕轨视频
- 导出烧录版视频
- 导出语言内容

## 产品结论

这项功能的产品结论如下：

1. 系统不应只服务英语学习场景。
2. 系统应支持“任意源语言 -> 任意学习目标语言”的学习模式。
3. 点句解析和高级翻译应以用户的解释语言输出。
4. 软字幕视频是首选导出方案。
5. 烧录版视频作为兼容性导出能力保留。
6. 导出能力至少覆盖：
   - 英文字幕版
   - 中文字幕版
   - 双语字幕版
   - 带字幕轨视频
   - 烧录版视频

这套设计会让应用从“英语视频学习工具”扩展为“多语言学习型字幕工作台”。
