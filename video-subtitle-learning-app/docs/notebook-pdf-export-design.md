# 收集册 PDF 导出设计

这份文档用于规划“词语册 / 句子册 PDF 导出”功能，目标是把当前电子收集册扩展成适合打印、复习和线下学习的纸质讲义。

## 1. 目标

当前项目已经支持收集册导出：

- `JSON`
- `CSV`
- `Markdown`

但这些格式更偏数据交换或文本整理，不适合直接打印学习。

PDF 导出的目标是：

- 支持用户把词语册、句子册导出为可打印讲义
- 输出浅色、低墨耗、适合黑白打印的版式
- 尽量保留学习信息层次，而不是简单把页面截图成 PDF
- 与当前桌面版 / Web 版共用同一套后端导出逻辑

## 2. 用户场景

### 2.1 词语册打印学习

用户希望把收集的词语打印出来，离线背诵、标注、默写。

典型内容包括：

- 词语
- 释义
- 备注
- 原句
- 学习语言句子
- 视频来源

### 2.2 句子册打印学习

用户希望把例句和解析打印出来，做精读、跟读、语法复习。

典型内容包括：

- 原句
- 学习语言句子
- 解析快照摘要
- 关键词
- 语法点
- 学习提示

### 2.3 教学 / 分享

用户可能把 PDF 发给同学、老师，或者直接打印成纸质资料。

因此 PDF 需要：

- 版式稳定
- 不依赖浏览器打印
- 黑白打印仍清晰

## 3. 设计原则

### 3.1 打印优先

第一版优先做“打印讲义版”，不是做“深色 UI 复刻版”。

因此默认版式采用：

- 白底或浅暖灰底
- 深灰 / 黑色文字
- 少量浅灰分隔线
- 不依赖彩色块表达重点

### 3.2 A4 优先

第一版默认：

- `A4`
- 纵向排版
- 适合家庭打印机和办公室打印

### 3.3 内容分层清晰

PDF 需要一眼看清：

- 收集册标题
- 语言方向
- 条目编号
- 重点字段
- 扩展字段

### 3.4 跨平台稳定

PDF 生成应由后端统一完成，不依赖用户浏览器打印。

这样能保证：

- Web 版和桌面版输出一致
- 不受浏览器分页差异影响
- 更适合便携版部署

## 4. 技术方案

## 4.1 方案选择

第一版推荐使用：

- 后端生成 PDF
- Python `reportlab`

原因：

- 当前后端已经是 `FastAPI + Python`
- 收集册数据已经集中在后端和数据库
- 不需要引入额外浏览器打印链路
- 更适合桌面便携版

## 4.2 为什么不优先用浏览器打印

浏览器打印或 HTML 转 PDF 也能实现，但第一版不优先采用，原因包括：

- 版式和分页更难控
- 深浅主题切换更麻烦
- 中文字体和多语言打印一致性更脆弱
- 桌面便携版中依赖更多

## 4.3 第一版核心依赖

- `reportlab`

后续如果需要：

- `pypdf`
- `pdfplumber`
- `poppler` 用于渲染检查

但第一版导出功能本身只强依赖 `reportlab`。

## 5. 字体策略

PDF 要支持中文，同时尽量兼顾英语、日语、韩语等学习内容。

第一版策略：

- Windows 环境优先尝试系统字体，例如 `Microsoft YaHei`
- 找不到时再回退内置 CID 字体

第一版先以“Windows 桌面便携版可用”为优先目标。

后续更稳妥的长期方案：

- 在项目中内置一套可再分发字体
- 例如 `Noto Sans SC` / `Source Han Sans`
- 打包时一起带进便携版

## 6. 第一版输出内容

## 6.1 词语册 PDF

建议每条词语包含：

- 编号
- 词语
- 释义
- 备注
- 原句
- 学习语言句子
- 视频标题
- 时间点

版式上：

- 词语和释义最突出
- 原句和学习语言句子作为例句区域
- 视频与时间作为元信息放在底部

## 6.2 句子册 PDF

建议每条例句包含：

- 编号
- 原句
- 学习语言句子
- 视频标题
- 时间点
- 解析快照摘要

解析快照摘要第一版先选这些字段：

- 优化译文
- 句子结构
- 学习提示
- 关键词

这样能控制页长，避免第一版过重。

## 7. 版式设计

## 7.1 页眉

每页顶部显示：

- 收集册名称
- 收集册类型
- 语言方向

## 7.2 页脚

每页底部显示：

- 页码
- 导出时间

## 7.3 颜色

推荐：

- 页面背景：白色
- 卡片背景：极浅灰
- 标题文字：深灰 / 黑色
- 标签文字：中灰
- 分隔线：浅灰

## 7.4 间距

要适合打印批注，因此应保留足够留白：

- 条目间距明显
- 段落行距略大于网页端
- 避免太紧凑的卡片堆叠

## 8. 接口设计

延续当前收集册导出接口：

- `GET /api/notebooks/{notebook_id}/export?format=pdf`

这样前端只需新增一个 `PDF` 选项，不需要新增单独入口。

## 9. 第一版实现范围

第一版只做下面这些：

1. 支持词语册导出 PDF
2. 支持句子册导出 PDF
3. 默认 A4 纵向
4. 默认浅色打印版
5. 支持页眉、页脚、页码
6. 支持多页自动分页

第一版暂不做：

- 封面模板切换
- 留空默写版
- 双栏对照打印版
- 自定义纸张大小
- 自定义主题皮肤

## 10. 第二版可扩展方向

后续可以继续做：

- “打印留白版”
- “词义遮盖版”
- “译文遮盖版”
- “带复习打勾框”
- “按视频分组导出”
- “按时间范围导出”
- “按最近新增 / 最近复习导出”

## 11. 风险与注意事项

### 11.1 字体覆盖

如果只依赖系统字体，不同机器覆盖范围可能略有差异。

因此第一版虽然能先做出来，但长期仍建议内置字体资源。

### 11.2 分页长度

句子册中如果解析快照过长，单条内容可能跨页。

第一版应优先保证：

- 不裁切文本
- 自动分页稳定

### 11.3 黑白打印表现

不能依赖颜色表达层次，因此字段标题、字号、粗细、留白要足够清楚。

## 12. 最终建议

第一版直接采用：

- 后端 `reportlab` 生成 PDF
- 复用现有收集册导出接口
- 默认浅色 A4 打印版
- 优先支持 Windows 桌面环境字体

这是当前项目阶段最稳、改造最小、最适合快速落地的方案。
